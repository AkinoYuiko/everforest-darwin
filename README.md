# everforest-darwin

A small, personal-use-first Home Manager module that applies a static [Everforest](https://github.com/sainnhe/everforest) palette to terminal applications. It is developed and tested on Apple Silicon macOS.

## Requirements

- Nix flakes
- Home Manager
- a truecolor terminal

Only `aarch64-darwin` is tested. The module does not reject other platforms, but they are not supported or covered by CI.

The module does not configure macOS appearance, terminal profiles, fonts, icons, wallpapers, or application installation.

## Installation

Add the flake input and make it follow the inputs already used by your configuration:

```nix
inputs.everforest = {
  url = "github:AkinoYuiko/everforest-darwin";
  inputs.nixpkgs.follows = "nixpkgs";
  inputs.home-manager.follows = "home-manager";
};
```

Import the module:

```nix
imports = [
  inputs.everforest.homeManagerModules.default
];
```

Enable the default `dark-medium` variant:

```nix
everforest.enable = true;
```

Override it when needed:

```nix
everforest = {
  enable = true;
  mode = "light";
  contrast = "soft";
};
```

`mode` accepts `"dark"` or `"light"` and defaults to `"dark"`. `contrast` accepts `"hard"`, `"medium"`, or `"soft"` and defaults to `"medium"`.

## Supported applications

The theme follows Home Manager's existing enable options and never enables an application itself.

| Application | Behavior |
|---|---|
| Bat | Installs and selects the generated `everforest` syntect theme |
| Btop | Installs the `everforest` theme, inherits the terminal background, and disables gradients |
| Fzf | Sets structured truecolor roles through `programs.fzf.colors` |
| Starship | Installs and activates the `everforest` custom palette without changing prompt layout |
| Tmux | Provides palette variables and color-only styles; the consuming configuration owns layout and behavior |

Fastfetch, Git, LSD, Yazi, Neovim, Lazygit, and Delta are not themed.

## Important boundaries

### Terminal background

Application base backgrounds inherit the terminal. Configure the Terminal.app profile separately and keep it consistent with `everforest.mode` and the selected contrast.

The module does not set `COLORTERM` or manage a Terminal.app profile.

### Color scope

Every Hex value directly generated by this module comes from the selected static palette. Application runtime effects, defaults, interpolation, command output, and explicit user overrides are outside that guarantee.

The module owns the supported applications' color fields but does not use `lib.mkForce`. Remove old hard-coded colors when migrating:

- remove Btop's existing `color_theme` and inline theme;
- remove Fzf `--color=...` entries from `defaultOptions`;
- replace fixed Starship color numbers such as `218` with a palette name such as `purple`;
- replace literal Tmux theme colors with the adapter-provided `#{@everforest_*}` variables.

### Tmux ownership

The Tmux adapter owns only colors: it defines `@everforest_*` variables from the selected palette and sets color-only status, window, pane, message, and clock styles. It does not set status layout, lengths, formats, Powerline glyphs, `terminal-features`, or other behavior.

Keep `status-left`, `status-right`, window formats, separators, font styling, and terminal capability declarations in the consuming Home Manager configuration. Layout strings can reference variables such as `#{@everforest_bg0}` and automatically follow `everforest.mode` and `everforest.contrast`.

### Starship names

The custom palette provides the normal Starship color names and these additional names:

```text
orange aqua accent muted success warning error info
```

Bright names map to existing Everforest colors; no derived bright colors are generated.

## Palette maintenance

`palette/data.nix` is the canonical, manually maintained static snapshot. Normal use and testing do not query Everforest upstream, and this repository provides no palette check or update command.

When an update is actually needed, open the upstream Everforest palette source, compare it manually, and directly edit `palette/data.nix`. Keep all six variants structurally complete and retain lowercase six-digit Hex values.
